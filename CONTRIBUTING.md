# Contributing to Ravenox 🐦‍⬛

Thank you for your interest in contributing to **Ravenox**, the sovereign intelligence system. We aim to maintain a high bar for performance, security, and architectural purity.

## 🏛️ Project Governance
Ravenox is a strictly directed project maintained by **Imran Shiundu (Lead Architect)**. All major architectural decisions and feature merges are governed by the Lead Architect.

## 🛠️ How to Contribute
1. **Report Issues**: If you find a bug or have a feature request, please open a [GitHub Issue](https://github.com/imranshiundu/Ravenox/issues).
2. **Submit PRs**: We welcome pull requests for:
   - Performance optimizations.
   - New Messaging channel integrations.
   - Skill refinements.
3. **Standards**: 
   - Code must be clean, typed, and efficient.
   - Follow the "Ghost Architecture" principles (no unnecessary polling or resource consumption).
   - **Crucial**: Run the validation suite (`./scripts/validate.sh`) and ensure it passes before submission.
   - Ensure all tests pass (`pnpm test`) before submission.

## 🤝 AI-Assisted Philosophy
We embrace AI-assisted engineering. If your contribution was developed with the help of LLMs (Claude, GPT, etc.), please disclose this in your PR description. Transparency helps our reviewers focus on structural integrity.

## 📞 Support
For complex architectural discussions or private inquiries, reach out to the Lead Architect via GitHub ([@imranshiundu](https://github.com/imranshiundu)).
